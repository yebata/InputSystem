Detailed Description

Input functions that have to be performed by the underlying input runtime.

The runtime owns the input event queue, reports device discoveries, and runs periodic updates that flushes out events from the queue. Updates can also be manually triggered by calling Update.

Property Documentation

◆ currentTime

double currentTime
get

The current time on the same timeline that input events are delivered on.

This is used to timestamp events that are not explicitly supplied with timestamps.

Time in the input system progresses linearly and in real-time and relates to when Unity was started. In the editor, this always corresponds to EditorApplication.timeSinceStartup.

Input time, however, is offset in relation to Time.realtimeSinceStartup. This is because in the player, Time.realtimeSinceStartup is reset to 0 upon loading the first scene and in the editor, Time.realtimeSinceStartup is reset to 0 whenever the editor enters play mode. As the resetting runs counter to the need of linearly progressing time for input, the input system will not reset time along with Time.realtimeSinceStartup.

◆ onDeviceDiscovered

Action<int, string> onDeviceDiscovered
set

Set delegate to be called when a new device is discovered.

The runtime should delay reporting of already present devices until the delegate has been put in place and then call the delegate for every device already in the system.

First parameter is the ID assigned to the device, second parameter is a description in JSON format of the device (see InputDeviceDescription.FromJson).

Mask that determines which input updates are executed by the runtime.
